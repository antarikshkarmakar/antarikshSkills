# Skill Observations

Backlog for improving framework skills from real work. Record only reusable process insights, not project secrets or one-off task details.

## Status Key
- OPEN
- ACTIONED
- DECLINED

## Public Safety Rule
Public-safe observations must not include client or project names, proprietary URLs, internal terms, credentials, personal data, or traceable examples. If unsure, mark `Type: internal`.

## Archive Rule
Keep all `OPEN` observations in this active file. During `/ak-compact`, if this file exceeds 150 lines or contains more than 20 `ACTIONED`/`DECLINED` entries, move `ACTIONED`/`DECLINED` entries older than 30 days to `memory/skill-observations.archive.md`. Do not archive `OPEN` entries. `/ak-skillset` reads this active file by default; consult the archive only when the user asks for older history.

## Observation Template

### Observation N: <short title>
**Status:** OPEN
**Date:** YYYY-MM-DD
**Skill:** ak-<name> | New skill candidate: <name> | All skills
**Type:** public-safe | internal
**Issue:** What repeated miss, edge case, dependency gap, or process failure appeared?
**Suggested improvement:** What small skill/ruleset change would prevent it next time?
**Principle:** What reusable rule should future agents remember?
